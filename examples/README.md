# Template Copilot Examples

This directory contains example projects demonstrating how to use the Template Copilot template effectively.

## Available Examples

### [sample-project](sample-project/) - TaskFlow Task Manager

A complete example of a fictional task management application called "TaskFlow". This demonstrates:

- ✅ All placeholders properly replaced with project-specific content
- ✅ Well-structured project overview, roadmap, and backlog
- ✅ Realistic feature descriptions and development milestones
- ✅ Professional documentation that passes validation
- ✅ GitHub issue references and team collaboration elements

**Start here** if you want to see what a finished project looks like after using the template.

## How to Use These Examples

### 1. Study the Structure

Compare the example files with the template files to understand:
- How to replace placeholders effectively
- How to structure your features and roadmap
- How to organize backlog items with GitHub issues
- Best practices for technical documentation

### 2. Copy What You Need

Feel free to:
- Copy sections that match your project type
- Adapt the writing style to your needs
- Use the structure as a starting point
- Reference the examples when customizing your own project

### 3. Learn by Comparison

Use the examples to:
- Understand the difference between template and final content
- See how different sections connect together
- Learn effective documentation patterns
- Validate your own documentation approach

## Creating Your Own Example

If you'd like to contribute an example:

1. Create a new directory for your example project
2. Fill in all template files with realistic content
3. Ensure validation passes: `python ../../validate_template.py`
4. Add a README explaining your example
5. Submit a pull request

We welcome examples for different project types:
- Web applications
- Libraries/frameworks
- CLI tools
- Mobile apps
- Data science projects
- DevOps tools

## Tips for Using Examples

1. **Don't copy blindly** - Adapt the content to your actual project
2. **Focus on the structure** - Learn the organization, not just the words
3. **Maintain authenticity** - Make sure your docs reflect your actual project
4. **Iterate** - Documentation improves over time as your project evolves

## Validation

All examples should pass the validation script when run from the repository root:

```bash
# From the template_copilot root directory
python validate_template.py
```

Note: The validation script checks the main template files (README.md, docs/), not the example projects. The examples are meant to show what completed documentation looks like after placeholders are replaced.
