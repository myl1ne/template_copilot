# Equipment Bug Fix

## Issue
Players were unable to equip items (like the rusty sword) purchased from the merchant. When trying to equip these items in the inventory, they would not move to the equipment slot.

## Root Cause
The bug was in the `buyItem` function in `world-rpg.js`. When a player buys an item from the merchant:

1. Items generated by the `LootSystem` have the `slot` property as a **direct property** of the item object:
   ```javascript
   {
     id: 'rusty_sword_common_123',
     name: 'Rusty Sword',
     type: 'weapon',
     slot: 'weapon',  // ← Direct property
     stats: { damage: 10, attackSpeed: 1.0 }  // ← stats doesn't contain slot
   }
   ```

2. The `Item` constructor expects the slot to be in the `stats` object:
   ```javascript
   constructor(id, name, icon, type, value, stats = {}) {
     // ...
     this.slot = stats.slot || null;  // ← Looks for slot in stats
   }
   ```

3. When buying an item, the old code created a new Item without preserving the slot property:
   ```javascript
   // OLD CODE (buggy)
   const newItem = new Item(item.id, item.name, item.icon, item.type, item.value, item.stats);
   // Result: newItem.slot = null (because stats.slot doesn't exist)
   ```

4. The `useItem` function checks for `item.slot` to equip items:
   ```javascript
   if (item.slot) {
     // Equip item
   }
   ```

Since `newItem.slot` was `null`, items could not be equipped.

## Solution
Modified the `buyItem` function to preserve all necessary properties from the original item:

```javascript
function buyItem(item) {
  const price = Math.ceil(item.value * 1.5);
  if (playerInventory.removeGold(price)) {
    // Create new item and preserve all properties from the original item
    const newItem = new Item(item.id, item.name, item.icon, item.type, item.value, item.stats);
    
    // Preserve slot property for equipment items
    if (item.slot) {
      newItem.slot = item.slot;
    }
    // Preserve rarity information
    if (item.rarity) {
      newItem.rarity = item.rarity;
      newItem.rarityColor = item.rarityColor;
      newItem.rarityIcon = item.rarityIcon;
    }
    // Preserve description
    if (item.description) {
      newItem.description = item.description;
    }
    
    // ... rest of the function
  }
}
```

## Testing
A test has been added in `tests/test-equipment-bug-fix.js` that:
1. Simulates an item from the merchant (with slot as direct property)
2. Demonstrates the old buggy behavior (slot is null)
3. Demonstrates the new fixed behavior (slot is preserved)
4. Verifies the item can pass the equipment check in `useItem`

Run the test:
```bash
node tests/test-equipment-bug-fix.js
```

## Impact
- Players can now successfully equip items purchased from merchants
- Item rarity information is also preserved (previously lost)
- Item descriptions are preserved
- No changes to the Item class or LootSystem were needed
- Minimal code change focused on the specific bug
